# Roles and Permissions

Nimbus controls what members can do through roles. There are four built-in roles,
and custom roles on the Growth and Enterprise plans.

**Built-in roles:**

- **Owner** — full control, including billing and deleting the workspace. There is
  always at least one owner; ownership can be transferred.
- **Admin** — manage members, projects, and workspace settings, but not billing
  unless also granted the Billing Admin role.
- **Member** — create and edit projects and tasks they have access to.
- **Guest** — view and act only within projects they're explicitly added to.

**Billing Admin** is an add-on permission that can be granted to any Admin so they
can manage payment methods and invoices without being an Owner.

**Custom roles (Growth+).** Create roles with granular permissions — for example a
"Project Lead" who can manage members within their projects but not workspace
settings. Configure these under **Settings → Roles**.

**Project-level access.** Independent of workspace role, each project can be set to
workspace-visible or restricted to specific members and guests. Use restricted
projects for sensitive work. Permission changes take effect immediately.
