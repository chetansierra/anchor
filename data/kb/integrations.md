# Integrations (Slack, GitHub, Jira)

Nimbus connects to the tools your team already uses. Manage all integrations under
**Settings → Integrations**.

**Slack** (all plans). Post task updates to a channel, create a Nimbus task from a
Slack message with `/nimbus`, and get a personal daily digest. Connect by clicking
**Add to Slack** and authorizing the workspace.

**GitHub** (Growth+). Link pull requests and commits to tasks by referencing the
task ID (e.g. `NIM-123`) in the PR title or commit message. The task shows PR
status and moves to **Done** automatically when the PR merges, if you enable that
rule.

**Jira** (Growth+). Two-way sync between Nimbus tasks and Jira issues, with field
mapping you control. Useful when engineering lives in Jira but the wider team plans
in Nimbus.

**Zapier and Make** (Growth+). Trigger and action support for no-code automations,
in addition to the native webhooks described in *API and Webhooks*.

**Calendar.** Subscribe to a project's due dates as a read-only iCal feed from the
project menu.

Each integration can be removed at any time; removing one stops syncing but never
deletes existing Nimbus data.
